# 🐴 API Routes Documentation

## Обзор

Приложение использует **Server-Side Rendering** с Jinja2 шаблонами. Все routes возвращают HTML-страницы, а не JSON.

---

## 📍 Основные Routes

### 1. **Главная страница**

#### `GET /`
- **Описание**: Главная страница приложения
- **Возвращает**: HTML-страница со списком всех скачек
- **Параметры**: Нет
- **Шаблон**: `index.html`

---

## 👤 Владельцы лошадей

### 2. **Список владельцев**

#### `GET /owners`
- **Описание**: Отобразить всех владельцев лошадей
- **Возвращает**: HTML-страница с таблицей владельцев и количеством их лошадей
- **Параметры**: Нет
- **Шаблон**: `owners.html`

### 3. **Добавить владельца**

#### `GET /owners/add`
- **Описание**: Форма для добавления нового владельца
- **Возвращает**: HTML-форма добавления владельца
- **Шаблон**: `add_owner.html`

#### `POST /owners/add`
- **Описание**: Обработка добавления нового владельца
- **Параметры формы**: 
  - `name` (string, обязательный) - Имя владельца
- **Валидация**:
  - Имя не может быть пусто
  - Имя должно быть уникально
- **Возвращает**: Редирект на `/owners` при успехе или обратно на форму при ошибке
- **Flash-сообщения**: Success/Error уведомления

---

## 🐎 Лошади

### 4. **Список лошадей**

#### `GET /horses`
- **Описание**: Отобразить всех зарегистрированных лошадей
- **Возвращает**: HTML-страница с таблицей лошадей (имя, владелец, рейтинг, количество скачек)
- **Параметры**: Нет
- **Шаблон**: `horses.html`

### 5. **Добавить лошадь**

#### `GET /horses/add`
- **Описание**: Форма для добавления новой лошади
- **Возвращает**: HTML-форма с выбором владельца
- **Требования**: Должны существовать владельцы
- **Шаблон**: `add_horse.html`

#### `POST /horses/add`
- **Описание**: Обработка добавления новой лошади
- **Параметры формы**:
  - `name` (string, обязательный) - Имя лошади
  - `owner_id` (integer, обязательный) - ID владельца
  - `rating` (float, опционально) - Начальный рейтинг (по умолчанию 5.0)
- **Валидация**:
  - Имя лошади обязательно
  - owner_id обязательно
  - Имя должно быть уникально
  - Рейтинг в диапазоне 0-10
- **Возвращает**: Редирект на `/horses` при успехе
- **Flash-сообщения**: Success/Error уведомления

## 🏁 Скачки

### 8. **Список скачек**

#### `GET /races`
- **Описание**: Отобразить все скачки
- **Возвращает**: HTML-страница с таблицей скачек (дата, количество участников, статус, действия)
- **Статусы скачки**:
  - `Scheduled` - Запланирована (результатов нет)
  - `In Progress` - Идёт (частичные результаты)
  - `Finished` - Завершена (все результаты записаны)
- **Параметры**: Нет
- **Шаблон**: `races.html`

### 9. **Главная страница скачек** (альтернатива)

#### `GET /` (также используется как главная)
- **Описание**: Главная страница с последними скачками
- **Возвращает**: HTML с отсортированным списком скачек по дате
- **Шаблон**: `index.html`

### 10. **Создать скачку**

#### `GET /races/add`
- **Описание**: Форма для создания новой скачки
- **Возвращает**: HTML-форма с выбором лошадей, жокеев и даты
- **Требования**: Должны существовать лошади и жокеи
- **Функции**:
  - Динамическое добавление участников (JavaScript)
  - Поддержка до 10 участников
- **Шаблон**: `add_race.html`

#### `POST /races/add`
- **Описание**: Обработка создания новой скачки
- **Параметры формы**:
  - `date` (datetime, обязательный) - Дата и время скачки в формате ISO 8601
  - `horse_0` до `horse_9` (integer) - ID лошади участника
  - `jockey_0` до `jockey_9` (integer) - ID жокея участника
- **Валидация**:
  - Дата скачки обязательна
  - Минимум один участник (пара лошадь-жокей)
  - Уникальность пары лошадь-жокей в рамках одной скачки
- **Возвращает**: Редирект на `/races` при успехе
- **Flash-сообщения**: Success (с количеством добавленных участников) / Error

### 11. **Детали скачки**

#### `GET /races/<race_id>`
- **Описание**: Просмотр подробной информации о конкретной скачке
- **Параметры URL**:
  - `race_id` (integer) - ID скачки
- **Возвращает**: 
  - HTML-страница с датой скачки, таблицей участников, результатами и рейтингами
  - 404 если скачка не найдена
- **Информация в таблице**:
  - Место финиша (с медалями 🥇🥈🥉)
  - Имя лошади и её владельца
  - Имя жокея
  - Рейтинги лошади и жокея
- **Шаблон**: `race_detail.html`

### 12. **Редактировать результаты скачки**

#### `GET /races/<race_id>/edit-results`
- **Описание**: Форма для редактирования результатов (мест) скачки
- **Параметры URL**:
  - `race_id` (integer) - ID скачки
- **Возвращает**: HTML-форма для ввода мест финиша
- **Шаблон**: `edit_race_results.html`

#### `POST /races/<race_id>/edit-results`
- **Описание**: Сохранка результатов скачки
- **Параметры формы**:
  - `place_<entry_id>` (integer) - Место финиша для каждого участника
- **Валидация**:
  - Места должны быть целыми числами
  - Места опциональны (могут быть пустыми)
- **Возвращает**: Редирект на `/races/<race_id>` при успехе
- **Flash-сообщения**: Success / Error уведомления

## ❌ Обработка ошибок

- **Возвращает**: HTML 404 страница с ссылкой на главную
- **Шаблон**: `404.html`

#### Внутренние ошибки
- **Описание**: Обработка неожиданных ошибок на сервере
- **Возвращает**: HTML 500 страница с ссылкой на главную
- **Шаблон**: `500.html`

## 🔄 Data Flow

### Добавление владельца:
```
GET /owners/add → Форма
↓
POST /owners/add → Валидация → БД → Редирект на /owners
```

### Добавление лошади:
```
GET /horses/add → Форма с списком владельцев
↓
POST /horses/add → Валидация → БД → Редирект на /horses
```

### Создание скачки:
```
GET /races/add → Форма с лошадями и жокеями
↓
POST /races/add → Валидация → БД → Редирект на /races
↓
GET /races/<id> → Просмотр деталей
↓
GET /races/<id>/edit-results → Форма результатов
↓
POST /races/<id>/edit-results → Обновление мест → Редирект на /races/<id>
```

## 📊 Связи между моделями

```
Owner (1) ──→ (Many) Horse
                    ↓
Jockey (Many) ──→ RaceEntry ←─── Race (1)
Horse (Many) ──→ RaceEntry ←─── (Many)
```

### RaceEntry (соединительная таблица):
- Хранит связь между Race, Jockey и Horse
- Содержит поле `place` для результатов

---

## 📝 Примечания

- Все routes используют **Jinja2 шаблоны**, HTML не возвращается через JSON
- Валидация выполняется на обеих сторонах: клиент (HTML5) и сервер (Python)
- Flash-сообщения выводятся в шаблонах с цветовой подсветкой
- Динамическое добавление участников скачки реализовано на JavaScript
- Все имена (лошадей, жокеев, владельцев) должны быть уникальны в БД
- Одна пара (лошадь + жокей) не может быть добавлена дважды в одну скачку
